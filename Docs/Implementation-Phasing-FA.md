# فازبندی اجرایی ELORIA - شروع از لاگین

مرجع طراحی برای تمام فازها `Design/design1.html` است. هیچ صفحه ای نباید از زبان بصری آن خارج شود: پس زمینه `#f2ecfb`، قاب مرکزی `#ece2f5` با stroke بنفش `#7850cd`، لوگوی italic serif، نقطه plum و مینیمالیسم شدید.

## فاز ۱ - Login First

**هدف:** اولین تجربه کاربر ورود/ثبت نام باشد، نه فروشگاه شلوغ.

- ساخت صفحه ورود و ثبت نام در `index.html`.
- استفاده از قاب مرکزی فایل `Design/design1.html` به عنوان کانتینر فرم.
- نگه داشتن لوگوی `Eloria` و نقطه پایین مطابق مرجع.
- ذخیره mock کاربر در `localStorage` فقط برای نمونه اولیه.
- اعتبارسنجی ساده ایمیل و رمز عبور.

**خروجی:** کاربر بتواند بین ورود و ثبت نام جابه جا شود و فرم را ارسال کند.

**وضعیت فعلی:** در `index.html` پیاده شده است. صفحه همیشه از لاگین شروع می شود و بعد از ورود نمونه، landing فروشگاه نمایش داده می شود.

## فاز ۲ - Account Shell

**هدف:** بعد از ورود، کاربر وارد یک پنل خیلی مینیمال شود.

- نمایش نام کاربر، ایمیل و دکمه خروج.
- ساخت لینک های اولیه: فروشگاه، علاقه مندی، سفارش ها.
- حفظ همان قاب/لوگو و رنگ های مرجع.

## فاز ۳ - Landing فروشگاه

**هدف:** بعد از auth، landing فروشگاه لوازم آرایشی ساخته شود.

- هرو مینیمال با همان فرم هندسی `design1.html`.
- معرفی دسته ها: آرایش، مراقبت پوست، عطر، ابزار.
- بدون عکس و تزئین اضافه تا زمانی که asset واقعی یا imagegen تایید شود.
- اگر asset تصویری لازم شد، فقط با اسکیل `imagegen` و بدون لوگو/برند جعلی ساخته شود.

**وضعیت فعلی:** نسخه اولیه landing بعد از login در `index.html` وجود دارد و از قاب مرکزی `Design/design1.html` برای هرو، دسته ها و کارت های محصول استفاده می کند.

## فاز ۴ - Product Catalogue

**هدف:** شبکه محصولات با همان زبان بصری.

- کارت محصول بسیار مینیمال داخل قاب های سفید/بنفش.
- قیمت فارسی، دسته، دکمه علاقه مندی و دکمه افزودن.
- فعلا دیتای mock؛ بعدا API.

## فاز ۵ - Cart و Checkout

**هدف:** جریان خرید کامل اما آرام و ساده.

- سبد خرید localStorage برای prototype.
- checkout مرحله ای: سبد، ارسال، پرداخت، تایید.
- همه CTAها بنفش و RTL.

## فاز ۶ - Backend واقعی

**هدف:** تبدیل prototype به محصول واقعی.

- Nuxt + MySQL + Prisma.
- auth امن با session httpOnly.
- جدول های users، products، carts، orders، reviews.
- migration و seed.

## فاز ۷ - Hardening و Launch

**هدف:** آماده سازی تولید.

- تست accessibility و responsive.
- تست E2E.
- امنیت، rate limit، backup و deploy.

## قانون اسکیل ها

- برای صفحات هندسی، فرم ها و UI اصلی از HTML/CSS مستقیم استفاده شود، چون مرجع `Design/design1.html` خودش SVG/HTML است.
- برای تصویر محصول، هرو تصویری یا asset رستری فقط زمانی از `imagegen` استفاده شود که واقعا به تصویر نیاز باشد و خروجی نباید لوگو، متن یا برند واقعی جعلی داشته باشد.
